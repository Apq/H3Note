# 英雄头像悬浮框绘制机制调查

## 目标
找出 HD Mod 或 SP 插件绘制英雄头像悬浮框的办法，用相同方法绘制远程对比框。

## 结论

### 英雄头像悬浮框确认是 HD Mod 自定义功能
- `heroWindowManager`（0x602770）在 0x60xxxx 范围 → **HD Mod 代码段**
- 原版 SoD 反编译源码中无此符号
- `LoadHeroPortraitImage`（0x4DE980）仅加载图片，未实现悬浮框

### 绘制机制（英雄头像 & 远程对比框 完全相同）
**都是直接绘制到 backbuffer（DirectDraw surface）：**

```
backbuffer = Lock()
DrawPcx16ToBackBuffer(pcx16, x, y)
Unlock()
H3Redraw(x, y, w, h)
```

具体函数：
- `DrawPcx16ToBackBuffer` — BattleValueInfo 自实现，等价于 H3API 的 `H3LoadedPcx16::DrawToBuffer`（0x44DF80）
- `DrawPcx16WithGDI` — GDI 备选方案（BattleValueInfo 自实现）
- `H3LoadedPcx16::DrawToBuffer`（0x44DF80）— H3API 标准封装

### RangedPanel 的 Hook 点（当前已变更）
| 地址 | 类型 | 作用 |
|------|------|------|
| 0x495C50 | HiHook | 战斗动画循环（**当前使用**） |
| 0x600430 | LoHook | backbuffer Blt 完成后（**已废弃**） |

### 当前 backbuffer 绘制方案

```cpp
// 1. 创建/重建合成图（尺寸匹配面板）
bg_composite_ = _Pcx16_::Create(panel_w, panel_h);

// 2. 把背景复制到合成图
memcpy(bg_composite_->buffer, bg_->buffer, panel_w * panel_h * 2);

// 3. 文字叠加到合成图
font->TextDraw(bg_composite_, tmp, left_col_x, row_y, col_w, row_h,
    NH3Dlg::eTextColor::WHITE, NH3Dlg::eTextAlignment::MIDDLE_RIGHT);

// 4. blt 到 backbuffer
DrawPcx16ToBackBuffer(bg_composite_, x, y, FALSE);

// 5. 强制刷新面板区域
o_WndMgr->H3Redraw(x, y, panel_w, panel_h);
```

**关键**：背景解码为 RGB565（16bpp），避免 HD/OpenGL BGRA backbuffer 颜色异常。

### 结论

当前实现使用合成缓冲方案（完全不等同英雄头像悬浮框）：

```
PCX (8bpp) → RGB565 _Pcx16_ (bg_) → bg_composite_
    ↓ 复制
    ↓ TextDraw() 文字叠加
bg_composite_ (背景+文字)
    ↓ DrawPcx16ToBackBuffer() blt
backbuffer
    ↓ H3Redraw() 刷新
屏幕
```

与英雄头像悬浮框的根本区别：
- 英雄头像：直接 blt PCX16 到 backbuffer，不处理文字
- 当前实现：背景+文字先在合成图合并，再一次性 blt
- Hook：当前用 `0x495C50`（HiHook，战斗动画循环）
