## 1. 项目目标与定位

H3Auto 是英雄无敌3（SoD + HD Mod）的战斗自动化 DLL 插件。核心目标：

**为人类玩家提供长时战斗的辅助，减少重复性操作。**

它需要能**代替玩家发出游戏操作**，实现途径有两种：

1. **直接调用原版函数** —— 写入 `BattleManager` 的 action 字段，交原版执行器完成动画、伤害、重绘、回合推进。
2. **模拟玩家操作** —— 从战场拾取坐标、模拟点击（如近战「站到某格再点某格」）。

### 1.1 核心原则

1. **只辅助人类玩家，绝不影响电脑阵营。** 回放/隐藏战斗（`IsHiddenBattle`）一律不接管。
2. **辅助而非接管。** 只生成动作和参数，不替换整个战斗循环；玩家未配置的部队保持手动。
3. **有限控制，不越权。** 对战争机器（弩车/箭塔/帐篷/投石车）只在原版本该自动操作的场景下提供「有限设置」，不提供直接完全控制。
4. **失败可降级。** 配置的行动无法实现时，若玩家允许则降级为自动防御；不允许则相当于放弃辅助（转为手动，交回玩家/原版）。
5. **配置仅驻留内存。** 5 套方案只存在进程内存，不写回 INI（INI 仅提供 UI 标签、热键等静态配置）。
6. **UI 只呈现合法组合。** 无效选项直接隐藏，不置灰。

## 2. 战争机器的条件化控制

英雄无敌3中，战争机器在英雄**缺少对应二级技能**时由电脑自动操作，且操作很笨。H3Auto 的定位是在这些场景下给玩家有限的设置能力，但**不提供直接完全控制**。

| 战争机器    | 相关技能                     | 有技能时                     | 无技能时（原版电脑操作，较笨）            |
| ------- | ------------------------ | ------------------------ | -------------------------- |
| 弩车 / 箭塔 | 炮术（Artillery, 技能索引 20）   | 原版按炮术逻辑，保持原样             | **允许玩家通过插件有限设置**（自动射击目标策略） |
| 急救帐篷    | 急救术（First Aid, 技能索引 27）  | 原版按急救术逻辑                 | **允许玩家有限设置**（类似弩车，选伤员策略）   |
| 投石车     | 弹道术（Ballistics, 技能索引 10） | **有弹道术时允许玩家设置**（选攻击城墙位置） | 原版电脑操作                     |
| 弹药车     | ——                       | 始终原版                     | 始终原版                       |

> 注意投石车与弩车/帐篷**方向相反**：投石车是「有弹道术才允许设置」，弩车/帐篷是「没有对应技能才允许设置」。原因是弹道术让玩家能手动瞄准投石车，而炮术/急救术会让弩车/帐篷进入原版自动逻辑。

「有限设置」= 只能设定目标选择策略（随机 / 数量优先等），不能像普通部队那样任意指定行动类型。

## 3. 需要提供的辅助能力

### 3.1 部队行动前英雄自动施法（快捷施法 / 循环施法）

- 每支部队行动前，英雄可先施放一个快捷施法法术。**循环施法只通过快捷施法功能实现** —— 施法内容就是英雄已设置的快捷施法快捷键（1-9,0，共 10 个）。
- **每回合施放的法术可能不同** —— 需要支持「循环施法」：按回合轮换一组快捷键，每个列表元素是一个快捷施法快捷键（1-9/0）。
- **UI 形态（已定）**：勾选「先快捷施法」后出现一排槽位小格 `[1▾][2▾]…` + 「＋」按钮，**最多 10 个**（对应 10 个快捷键）；战斗时按回合推进游标轮换（末位回首位）。
- 当前 UI 已有单槽位雏形（`quickCastFirst` + `spellSlot` 1-9/0），**现状**：尚未实现「每回合轮换」的循环列表，也未接执行端。

### 3.2 自动防御

- 部队轮到行动时直接防御。无目标，最简单的辅助。

### 3.3 自动射击（远程攻击）

面向远程部队（含无炮术的弩车/箭塔）。目标选择策略：

- **随机目标**
- **远程速度优先**（优先打对方远程单位，其次高速单位）——**现状**：`SEL_RANGED_SPEED` 尚未实现真实排序，暂等价随机。
- **数量优化**（数量最多 / 数量最少）

### 3.4 近身攻击

- 需要能设置「**从哪个站立格，攻击哪个格子**」（模拟玩家「走到 standHex 再点 attackHex」）。
- 两个格子都通过**点击进战场拾取**方式设置（与循环移动/循环施法同一交互风格），不再用坐标下拉：
  - 点「站立格」按钮 → 进战场拾取，点战场任意格作为站立格。
  - 点「攻击格」按钮 → 进战场拾取，点站立格的相邻格作为攻击格。
- 攻击格上有敌人（头格或双格尾格）时才提交近战；否则视为无法实现。
- 站立格候选 = 全战场；攻击格候选 = 站立格的相邻格。

### 3.5 循环移动

- 设定一个**有序路径点列表**（最多约 6 个），部队逐点巡逻循环（末点回首点）。
- 路径点用**「点击＋追加」序列式**交互（与循环施法槽位一致）：UI 显示已选路径点 `[A01][B05]…＋`，点末尾的「＋」进入战场拾取、选一格即追加一个点，点某个已选点可移除。免手写、可增量编辑。

### 3.6 降级为自动防御

- 上述主动行动（射击/近战/移动/施法）在某回合无法实现时：
  - 玩家**勾选允许降级** → 转为自动防御，辅助继续。
  - 玩家**未勾选** → 放弃本次辅助，相当于没有辅助（转手动，交回玩家操作）。
- 战争机器不适用降级（它们的「不执行」就是回到原版逻辑）。

## 4. 数据结构（源：`modules/ConfigLog.inc.cpp`）

### 4.1 枚举

```cpp
enum AutoActionKind : uint8_t {
    AA_MANUAL = 0,        // 手动（不干预）
    AA_DEFEND,            // 防御
    AA_WAIT,              // 等待（枚举保留，UI 下拉暂不提供）
    AA_MOVE,              // 循环移动
    AA_MELEE_ATTACK,      // 近战攻击
    AA_RANGED_ATTACK,     // 远程攻击（自动射击）
    AA_FIRST_AID,         // 急救帐篷疗伤
    AA_CATAPULT,          // 投石车攻击城墙
    AA_COUNT
};

enum AutoTargetKind : uint8_t { AT_NONE = 0, AT_STACK, AT_POSITION, AT_COUNT };

enum AutoTargetSide : uint8_t {
    ATS_OWN = 0, ATS_ENEMY, ATS_EITHER, ATS_COUNT
};

enum AutoTargetSelector : uint8_t {
    SEL_FIXED = 0, SEL_RANDOM, SEL_SEQUENTIAL, SEL_NEAREST, SEL_FARTHEST,
    SEL_RANGED_SPEED, SEL_COUNT_HIGH, SEL_COUNT_LOW, SEL_MOST_WOUNDED,
    SEL_NONE, SEL_COUNT
};
```

> **枚举稳定性约定**：枚举值一经分配不再移动；新增项一律追加末尾。UI 标签数组、INI `[Actions]/[Sides]/[Selectors]` 段严格**按枚举下标**排列；「下拉显示哪些项、什么顺序」由 `GetAllowed*` 构建器控制，与枚举下标解耦。（历史教训：INI 标签顺序停留旧模型导致下拉「近战攻击」实为「等待」。）

### 4.2 规则结构

```cpp
struct AutoTargetRule {
    AutoTargetKind     kind;
    AutoTargetSide     side;
    AutoTargetSelector selector;
    int8_t  fixedSide;          // 固定部队：0/1，-1=未设
    int8_t  fixedSlot;          // 固定部队：0..20，-1=未设
    int16_t fixedCreatureId;    // 跨战斗指纹，-1=未设
    int16_t fixedHex;           // 固定位置 hex 0..186，-1=未设

    int16_t meleeStandHex;      // 近战站立/接近格，-1=未设
    int16_t meleeAttackHex;     // 近战攻击点击格，-1=未设

    int16_t moveWaypoints[6];   // 循环移动路径点，原版 hex 1..185，-1=空
    int8_t  moveWaypointCount;  // 有效点数 0..6
    int8_t  moveWaypointCursor; // 运行时游标：下一个要走向的点索引
};

struct AutoStackRule {
    AutoActionKind action;
    AutoTargetRule target;
    bool    allowDefendFallback; // 允许降级为防御（仅普通部队）
    bool    quickCastFirst;      // 行动前先快捷施法
    int8_t  spellSlots[10];      // 循环施法快捷键序列（每个值 1-9/0），-1=空槽
    int8_t  spellSlotCount;      // 有效槽位数 0..10
    int8_t  spellCursor;         // 运行时游标：下一回合要施的槽位索引
};
```

> **循环施法**：列表最多 10 个快捷键（对应快捷施法的 10 个槽位 1-9/0），战斗中每回合按 `spellCursor` 轮换（末位回首位）。**现状**：代码目前仅单槽位 `spellSlot`，待扩为上述序列结构。

### 4.3 方案存储

```cpp
AutoStackRule g_profiles[5][21];   // 5 套已确认方案 × 21 槽位，仅驻留内存
int           g_active_profile;    // 当前生效方案序号
AutoStackRule g_active_rules[21];  // 运行时视图（执行端读取）
```

- 默认全为 `AA_MANUAL`（不干预）。

## 5. 行动可选集（源：`CellControl_GetAllowedActions`）

按单位类型 + 英雄技能过滤，只呈现合法行动：

| 单位类型    | 可选行动                 |
| ------- | -------------------- |
| 弹药车     | 手动（实际不进入面板）          |
| 急救帐篷    | 手动、急救治疗（无急救术时开放）     |
| 投石车     | 手动、投石车攻击（有弹道术时开放）    |
| 弩车 / 箭塔 | 手动、远程攻击（无炮术时开放）      |
| 普通远程部队  | 手动、防御、循环移动、近战攻击、远程攻击 |
| 普通近战部队  | 手动、防御、循环移动、近战攻击      |

- `is_ranged` 由面板打开时按 `stack.info.shooter` 精确判定并存入 `CellData.is_ranged`；切换方案时复用，不用 `creature_type` 粗判（否则射手如大法师丢失远程选项）。
- **现状**：行动可选集当前主要按单位类型过滤，战争机器的「技能条件」判定在**执行端**（`DecideTakeover`）做；UI 端是否也要按技能隐藏选项待定。

## 6. 目标可选集（源：`CellControl_GetAllowedSelectors`）

- **近战攻击**：不走选择器，固定 `SEL_FIXED`（站立格 + 攻击格）。
- **循环移动**：不走选择器，用路径点列表（`SEL_FIXED`）。
- **其它需要目标的行动**（远程/急救/投石）：统一提供 4 项 —
  `无(SEL_NONE) / 远程高速优先(SEL_RANGED_SPEED) / 数量最多(SEL_COUNT_HIGH) / 随机(SEL_RANDOM)`。

> 精简为 4 项符合「减少重复操作」定位，不追求覆盖全部 AI 策略。

---

## 7. 规则规范化（源：`CellControl_NormalizeRule`）

行动变化或加载方案时调用，保证规则自洽：

1. 若 `action` 不在单位允许集内，回退允许集首项（或 `AA_MANUAL`）。
2. 不需要目标的行动：清空目标区，`allowDefendFallback=false`。
3. 各行动设默认目标（`DefaultTargetForAction`）：近战 `AT_POSITION+SEL_FIXED`、远程 `AT_STACK+敌方`、急救 `AT_STACK+己方`、投石 `AT_POSITION+敌方`、移动 `AT_POSITION+SEL_FIXED`（路径点）。
4. **选择器收敛**：近战/移动固定 `SEL_FIXED`；其它行动若 `selector` 不在允许集则吸附首项。
5. 不显示降级复选框的行动：`allowDefendFallback=false`。

## 8. 设置面板 UI（源：`modules/SettingsDlg.inc.cpp` + `CellControl.inc.cpp`）

### 8.0 模态方案：加固伪模态（不改真模态）

> **决策依据**：原版真模态（`H3Dlg::vShowAndRun` 虚表 v18）跑一个**嵌套阻塞消息循环**，一旦启动就独占所有鼠标/键盘消息，下层战斗对话框（消息入口 `0x4746B0`）完全收不到消息 —— 战场被冻结，无法点选。而本插件的核心交互（近战两格、循环移动路径、循环施法槽位）**大量依赖战场拾取**，若用真模态，每次「点＋拾取」都要关窗→拾取→重开窗，闪窗且交互跳。故**放弃真模态**，改为**加固当前伪模态**。

伪模态机制（保留）：面板自绘到 `screenPcx16` + 自建鼠标 hook（`Hook_BattleMsgProc` 屏蔽 hover、`ForcePanelModalDepth_` 顶住模态计数器 0x69FEA4）。战斗消息循环仍在跑，所以拾取时点战场能被 hook 抓到转成 hex。

**加固目标**（拾取时确保背景战斗界面不响应误点）：

1. **面板展开时**：hook 拦截并吞掉所有落在战斗界面的鼠标点击（左右键 up/down），只放行落在面板矩形内的消息，杜绝误触底层战斗按钮/部队。
2. **进入战场拾取时**（`s_move_pick_cell` / `s_melee_pick_mode` 激活）：仅放行「战场空格点选」这一类点击用于取 hex，其余落在原战斗 UI 控件（施法书、投降、战斗设置等按钮）上的点击一律吞掉，防止拾取过程误触发原版功能。右键仅用于「结束拾取」，不透传给战斗界面。
3. **面板/拾取结束后**：恢复正常消息流，`ForcePanelModalDepth_(false)` 复位。

> 本质：伪模态靠 hook 精确控制「哪些点击透传、哪些吞掉」，达到接近真模态的输入隔断效果，同时保留战场拾取能力。**现状**：当前 hook 在拾取时已挂起面板并接收战场点击，但对「落在原战斗 UI 控件上的误点」尚未全面吞掉，需按上述加固。

### 8.1 布局：一行一格宽卡片（568×72）

单列，每支己方部队一张横排宽卡。根据单位类型分两种卡片：

**普通部队卡**：

```text
[图标]  先快捷施法 ☑ [1▾][2▾]…＋   行动 [远程攻击▾]      选择器 [数量最多▾]
 58×64  数量 12                       允许降级 ☑          （近战: 站立格/攻击格）
         位置 A07                                          （移动: 编辑路径 3/6）
```

**战争机器卡**（弩车/箭塔/帐篷/投石车，仅在技能条件满足时可设）：

```text
[图标]  数量 1        辅助 [启用☑]           选择器 [数量最多▾]
 58×64  位置 K08                             （投石: 编辑城墙目标）
```

- 战争机器卡**不显示施法两行、不显示「允许降级」复选框**（降级对战争机器无意义，它们“不执行”就是回到原版）。
- 行动不做下拉，改为一个「**启用/禁用**」开关：启用=插件按选择器接管它唯一能做的事（弩车/箭塔=远程攻击，帐篷=急救，投石车=投石）；禁用=交回原版（相当于「手动」）。
- 位置显示保留（战争机器位置虽固定，但对玩家仍有参考价值）。
- 图标（x=4，58×64）+ 金框（60×66，`HA_icon_frame.pcx`）。

### 8.2 普通部队卡小列布局

- **位置/数量小列**（x=68，宽 74）：顶部两行「先快捷施法」复选框 + 「选择魔法」快捷键序列（`[1▾][2▾]…＋`，最多 10）；下方数量（右对齐）、位置（贴格子下边缘左对齐）。
- **行动小列**（`CC_COL2_X`）：行动下拉（上）+「允许降级为防御」复选框（下）。
- **目标小列**（`CC_COL3_X`）：
  - 远程/急救/投石：选择器下拉（上）+ 固定说明（下，远程不显示说明）。
  - 近战：站立格（上）+ 攻击格（下），两格都通过**点击进战场拾取**设置（点该格按钮进拾取，点战场格回填），与循环移动/循环施法同一交互风格。
  - 循环移动：路径点序列 `[A01][B05]…＋`（点「＋」进战场拾取追加，点已选点移除），与循环施法同一交互风格。

### 8.3 下拉展开几何

每个下拉从其所在行下边缘展开（`GetExpandListLeftX/TopY/Width`）。绘制（`DrawDropdownItem` 接 `list_top_y/list_left_x`）与命中（`HitExpandVisibleIndex`）用同一基准，避免错位。魔法槽位下拉用窄列宽（`CC_POS_W`），其余用 `CC_COL_W`。

### 8.4 战场拾取（近战格 / 移动路径）

复用 `Hook_BattleMsgProc` 内的战场点击捕获 + `0x464380`（屏幕坐标→hex）转换。统一采用**「点击＋追加」**交互：

- 点序列末尾的「＋」→ 挂起面板输入，进入战场拾取模式。
- 每次左键点战场空格追加一个路径点（最多 6），实时回填重画。
- 满 6 自动结束；不满时右键结束（两种都支持）。
- 点已选路径点可移除该点。
- 近战站立/攻击格为**单发拾取**（点该格按钮进拾取，点战场格选中后即回填退出）：站立格可点全战场任意格，攻击格只认站立格的相邻格；拾取攻击格时点了非相邻格视为无效点（忽略，不回填、不退出拾取）。**待改**：当前近战主用坐标下拉，需切换到点击进战场拾取（复用 `s_melee_pick_mode` 机制并接线激活）。

### 8.5 标签来源（`LoadLabels_`）

从 `H3Auto.ini` 的 `[Actions]/[Sides]/[Selectors]` 按枚举下标读取，缺失用 `DEFAULT_*_LABELS` 兜底。

## 9. 方案保存流程

目标行为：打开界面自动加载上次设置；调整过程中不影响已保存设置；仅点「确定（勾✓）」才真正写入。

- 打开面板：`g_profiles → s_p.draft_rules`，进入草稿副本编辑，原方案不动。
- 编辑格子：改动只写 `s_p.cells[].data.rule`（内存草稿）。
- 切换方案（`SelectProfile_`）：先 `SaveCurrentCellsToDraft_` 存回草稿，再 `LoadSelectedProfileIntoCells_` 载入目标方案。
- 点确定（`CommitAndCloseSettingsPanel_`）：`SaveCurrentCellsToDraft_` → `CommitProfiles` 一次性提交全部 5 套到 `g_profiles`。
- 点取消 / ESC：直接关闭，草稿丢弃。

### 9.1 设置生命周期（锚定「一场战斗」）

设置（5 套方案）只对**当前这一场战斗**有效，且生命周期以「玩家是否接受战斗结果」为准，而非战斗动画是否播完。

关键点：**重打不算新战斗**。常见流程：

1. 战斗打完 → 弹战斗结果窗 → 玩家点「取消 / 重打」→ 这是**重打同一场**，5 套方案必须原样保留。
2. 重打完再弹结果窗 → 再取消重打 → 仍是同一场，设置照样保留。
3. 只有玩家**点「确定 / 接受」**结果，这场战斗才真正结束 → 此时才清除内存中的设置。

底层原因：不接受结果就一直能重打（除非读档），所以设置必须一直留着；一旦接受，就再也回不到这场战斗（想再打只能读档，那是另一次战斗上下文），设置才可以丢弃。

实现要点：

- 设置在**进入战斗**时初始化 / 沿用（同一场重打时不重置）。
- 清除时机挂在「玩家接受战斗结果」事件上——**待定位的 hook 点**：需逆向原版战斗结果窗口的「接受」与「取消/重打」分支，仅在「接受」分支清除 `g_profiles`。取消/重打分支绝不清除。
- **现状**：清除时机的 hook 点尚未定位，实现时再逆向确认。

## 10. 执行分发（源：`modules/AutoExecute.inc.cpp`）

### 10.1 接管判定（`DecideTakeover`）

回放/隐藏战斗（`IsHiddenBattle`）一律 `CD_KEEP_ORIGINAL`。否则：

| 单位    | 判定                                     |
| ----- | -------------------------------------- |
| 弹药车   | 保持原版                                   |
| 急救帐篷  | 配了急救→执行；否则有急救术→原版，无→交 AI               |
| 投石车   | 配了投石→执行；否则有弹道学→原版，无→交 AI               |
| 弩车/箭塔 | 配了远程→执行；否则有炮术→原版，无→交 AI                |
| 普通部队  | 手动→保持原版；否则按配置执行，绝不交回 AI（前提是当前为人类可交互回合，隐藏/回放战斗已在上面拦掉） |

英雄技能查询：`HeroHasSkill`（弹道术 10 / 炮术 20 / 急救术 27）。

### 10.2 动作提交（`TrySubmitConfiguredAction_`）

按 `rule.action` 分发到 `Submit*`；普通部队失败且允许时降级 `SubmitDefend_`：

- **防御** `SubmitDefend_`：`action=BA_DEFEND`。
- **等待** `SubmitWait_`：`action=BA_WAIT`。
- **循环移动** `SubmitMove_`：**现状**仍是旧的单目标移动（`ResolvePositionTarget_` → `BA_WALK`），**待改**为按 `moveWaypoints` + `moveWaypointCursor` 逐点巡逻循环。
- **近战** `SubmitMelee_`：攻击格上有敌人才出手；`move_type=7`，`action=BA_WALK_ATTACK`，param=站立格，target=攻击格。
- **远程** `SubmitRanged_`：`SelectStackTarget_` 选敌方目标，`action=BA_SHOOT`。
- **急救** `SubmitFirstAid_`：选己方伤员，`action=BA_FIRST_AID`。
- **投石** `SubmitCatapult_`：城墙位置，`action=BA_CATAPULT`。

写入后原版主循环（`FUN_004786b0` 等）接管动画/伤害/回合推进。

### 10.3 目标选择（`SelectStackTarget_` / `PickBySelector_`）

先按 `side` 过滤双方 `stack[2][21]` 得候选，再按选择器挑一个：

- `SEL_NEAREST/FARTHEST`：按 `hex_ix` 距离。
- `SEL_COUNT_HIGH/LOW`：按存活数量。
- `SEL_MOST_WOUNDED`：按损失生命 `(count_at_start-count_current)*1000 + lost_hp`。
- `SEL_SEQUENTIAL`：按 `army_slot_ix`。
- `SEL_RANGED_SPEED / SEL_RANDOM / 其它`：**现状**取随机候选。
- `SEL_FIXED`：固定部队（`fixedSide/fixedSlot/fixedCreatureId`）。

### 10.4 快捷施法执行

**现状**：`quickCastFirst / spellSlots` 尚未接执行端。目标：部队行动前，若勾选则先用英雄快捷施法施放当前游标指向的快捷键法术，再执行本部队的主动行动；循环施法每回合推进 `spellCursor`（末位回首位）。
